SSM框架(三)SpringMVC的应用一

  • Java
  • 2019-08-18

1. SpringMVC的介绍

SpringMVC全名是model view controller，从名称就可以看出他的意思，就是模型-视图-控制器三个部分，利用MVC可以将业务逻辑，数据、和界面显示分开。

M-Model 模型层：Model（模型）的主要职责是负责业务逻辑，其又包含两层：业务数据层（Bean，DAO，DAOImpl）和业务逻辑处理（service）。

V-View 视图层：view（视图）的主要职责负责界面显示和用户交互，通常视图是依据模型数据创建的，一般上来说就是jsp。

C-Controller 控制层：Controller（控制器） 是模型层M和视图层V的桥梁，控制器接受用户的输入并调用模型和视图去完成用户的需求，控制器本身不输出任何东西和做任何处理。它只是接收请求并决定调用哪个模型构件去处理请求，然后再确定用哪个视图来显示返回的数据。

Spring Web MVC核心组件：
DispatcherServlet（前端控制器，请求入口）
HandlerMapping（处理器映射器，请求派发）
Controller（处理器，请求处理流程）
ModelAndView（模型，封装业务处理结果和视图）
ViewResolver（视图，视图显示处理器）

2. SpringMVC的工作流程

首先由用户发送请求到前端控制器（DispatcherServlet），前端控制器能够接受、分发请求和响应结果，当他接受到前端请求的时候，根据请求的URL查找处理器，这里就用到了HandlerMapping，HandlerMapping的作用就是根据URL寻找处理器，HandlerMapping找到相应的处理器后并不是直接将结果返回给处理器，而是将结果先返回给前端控制器DispatcherServlet，由前端控制器再次分发结果到处理器适配器HandlerAdapter，处理器适配器将结果拿给处理器执行，处理器的作用是通过实现的接口处理业务请求，将执行结果返回给处理器适配器HandlerAdapter，这个时候处理器适配器HandlerAdapter，再将结果返回给前端控制器，前端控制器再请求视图解析器，视图解析器根据请求生成视图对象，最后交给前端控制器，完成视图渲染后返回给前端页面，这样一次完整的请求就结束了。

3. 基于注解的SpringMVC开发

基于注解的SpringMVC开发能够大大的减少我们的工作量，不需要再像上面一样去一一配置处理器映射器了，只需要在某些需要的类或者方法前面加上一些注解，再配置上注解扫描，SpringMVC就会直接给我们映射好。

修改applicationContext.xml文件

新建一个controller，IndexController，在类名前面加上@controller注解，@RequestMapping表示访问路径，如果不加，那我们访问的时候就直接写方法前面那个@RequestMapping的路径，如果加上了@RequestMapping，访问路径中也要加上。

新建一个index.jsp，然后启动服务器，访问localhost:8080/你的项目名/index/user.do

4. SpringMVC页面值的接收与传递

Spring Web MVC请求提交数据到控制器主要有以下三种方式
1、使用HttpServletRequest获取，例如：String userName=req.getParameter(“name”);
2、使用@RequestParam注解，Spring会自动将表单中参数注入到方法参数中（两个名称要一致）

3、使用自动机机制封装成Bean对象，实体类的属性名和form表单组件的name相同


当Controller组件处理后，需要向jsp页面传值，可以使用以下几种方式：

1、直接使用HttpRequestServlet 和 Session，例如：request.setAttribute(“message”, “注解controller访问成功！”);
2、使用map，Model，ModelMap, ModelAndView实现

在index.jsp中加一句：<h2>${username}</h2>

3、使用@ModelAttribute和@SessionAttributes注解
